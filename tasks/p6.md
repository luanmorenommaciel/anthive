# p6 — `swarm dispatch --cloud`

> **Why post-MVP:** cloud requires a separate Console API key and bills pay-per-token. Not on the critical path, but unlocks laptop-free / overnight / beyond-Max-quota workflows. Build after ACT paper v1 draft.

---

## Goal

Implement the `CloudDispatcher` class satisfying the `Dispatcher` interface from p3, using Anthropic's Managed Agents API (`managed-agents-2026-04-01` beta).

Flow:
1. `swarm dispatch --cloud T-X`
2. Create agent via `client.beta.agents.create()` — model + system prompt + tools
3. Create environment via `client.beta.environments.create()` — container config
4. Create session via `client.beta.sessions.create()` — agent + env + title
5. Send prompt as `user.message` event to `/sessions/{id}/events`
6. Stream events; update session log + emit OTEL spans
7. On `session.status_idle` event → reap files → commit to local worktree → mark READY-TO-MERGE

Key constraint: **requires separate `ANTHROPIC_API_KEY` (not Max 20x OAuth).** The dispatcher enforces this with a clear error if the key is missing.

---

## Success criteria

- [ ] `swarm dispatch --cloud T-X` creates a Managed Agents session and streams its output
- [ ] Session runs in Anthropic cloud (no local tmux, no local CPU)
- [ ] Files produced in the container are pulled back via the Managed Agents files API
- [ ] Pulled files are committed to a local git worktree branch (same shape as local mode for merger compatibility)
- [ ] Cost is tracked in Langfuse (same schema as local) — `$0.08/session-hour` runtime + tokens
- [ ] Budget cap enforcement: session auto-killed if spend exceeds `budget_cap_usd`
- [ ] Pre-flight check: `ANTHROPIC_API_KEY` in env OR `swarm.toml.dispatcher.cloud.api_key_env` points to a set var
- [ ] Pre-flight cost estimate shown before dispatch (with `--yes` to skip)
- [ ] `swarm watch` dashboard treats cloud sessions identically to local (same columns)
- [ ] Pytest: 6+ tests with mocked Anthropic SDK

## Implementation sketch

# agentswarm/dispatchers/cloud.py

import os, time
from pathlib import Path
from anthropic import Anthropic
from anthropic._exceptions import APIStatusError
from .base import Dispatcher, SessionHandle, AlreadyDispatchedError
from ..schemas import TaskFrontmatter, SessionLogFrontmatter
from ..state import write_session_log
from ..observability import session_span, emit_lifecycle_event

class CloudDispatcher(Dispatcher):
    """Dispatch via Anthropic Managed Agents cloud sessions."""

    def __init__(self, config):
        self.config = config
        api_key_env = config.get("api_key_env", "ANTHROPIC_API_KEY")
        if not os.getenv(api_key_env):
            raise RuntimeError(
                f"Cloud mode requires {api_key_env} in environment.\n"
                f"Get a key at https://console.anthropic.com/settings/keys\n"
                f"Note: cloud mode bills to Console credits, not your Max subscription."
            )
        self.client = Anthropic(
            api_key=os.getenv(api_key_env),
            default_headers={"anthropic-beta": "managed-agents-2026-04-01"},
        )
        self.budget_cap_usd = config.get("budget_cap_usd", 5.00)
        self.environment_id = config.get("environment_id")  # reuse if pre-created

    def dispatch(self, task, prompt, prompts_dir):
        slug = task.slug
        worktree = Path(self.config["worktree_dir"]) / slug
        branch = f"session/{slug}"

        # 1. Create local worktree (cloud dispatches still land PRs locally)
        import subprocess
        subprocess.run(["git", "worktree", "add", "-b", branch, str(worktree)], check=True)

        # 2. Resolve agent definition
        agent_def = load_agent_def(task.agent)

        # 3. Create Managed Agents agent
        agent = self.client.beta.agents.create(
            name=f"swarm-{task.id}",
            model=task.prefer_model or "claude-opus-4-7",
            system=agent_def.system_prompt,
            tools=[{"type": "agent_toolset_20260401"}],  # full default tool suite
        )

        # 4. Ensure environment (reuse if configured)
        env_id = self.environment_id
        if not env_id:
            env = self.client.beta.environments.create(
                name=f"swarm-env-{slug}",
                config={"type": "cloud", "networking": {"type": "unrestricted"}},
            )
            env_id = env.id

        # 5. Create session
        session = self.client.beta.sessions.create(
            agent=agent.id,
            environment_id=env_id,
            title=task.title,
        )

        # 6. Write local session log (so swarm watch sees it)
        session_log = Path("logs/sessions") / f"{slug}.md"
        fm = SessionLogFrontmatter(
            session_id=slug,
            task_id=task.id,
            slug=slug,
            branch=branch,
            worktree=str(worktree),
            container=session.id,  # cloud session ID
            mode="cloud",
            forked_from_sha=git_head_sha(),
            created=now_iso(),
            status="COOKING",
            last_heartbeat=now_iso(),
            last_note=f"dispatched to cloud, session {session.id}",
            primary_agent=task.agent,
            model=task.prefer_model or "opus",
            touches_paths=task.touches_paths,
        )
        write_session_log(session_log, fm, body_from_task(task))

        # 7. Start parent OTEL span
        with session_span(slug, task.id, task.agent, "cloud"):
            # 8. Send the user message
            self.client.beta.sessions.events.send(
                session.id,
                events=[{"type": "user.message", "content": [{"type": "text", "text": prompt}]}],
            )

            # 9. Start the streaming watcher in a background thread
            handle = SessionHandle(
                session_id=slug,
                tmux_pane=None,
                worktree=worktree,
                branch=branch,
                log_path=session_log,
                cloud_session_id=session.id,
            )
            self._spawn_watcher(handle, session.id)

        return handle

    def _spawn_watcher(self, handle, cloud_session_id):
        """Background thread: streams events, updates log, enforces budget."""
        import threading
        def watch():
            last_spend = 0.0
            with self.client.beta.sessions.events.stream(cloud_session_id) as stream:
                for event in stream:
                    if event.type == "agent.message":
                        self._append_to_log(handle, f"[{event.type}] {event.content[0].text[:200]}")
                    elif event.type == "agent.tool_use":
                        self._append_to_log(handle, f"[tool_use] {event.name}")
                    elif event.type == "session.status_idle":
                        self._update_status(handle, "READY-TO-MERGE", "session idle — ready to reap")
                        self._reap(handle, cloud_session_id)
                        break
                    # Budget enforcement
                    spend = self._get_current_spend(cloud_session_id)
                    if spend > self.budget_cap_usd * 1.2:  # 20% over cap = kill
                        self._update_status(handle, "BLOCKED", f"killed: exceeded budget ({spend:.2f})")
                        self.shutdown(handle)
                        break
        threading.Thread(target=watch, daemon=True).start()

## Do-not-touch list

- Don't mix subscription auth with cloud mode — cloud is API-key-only
- Don't remove local mode — cloud is additive, not a replacement
- Don't auto-retry on API errors blindly — exponential backoff with max 3 retries
- Don't exceed budget cap — kill sessions that do

## Budget safety

Three layers of defense:

1. **Pre-flight estimate** — before `swarm dispatch --cloud`, show estimated cost; require `--yes` to skip confirmation
2. **Per-session cap** — `budget_cap_usd` in config; kill session that exceeds 120% of cap
3. **Global daily cap** — `[dispatcher.cloud.daily_budget_usd]`; `swarm dispatch --cloud` refuses if today's spend already exceeds

Example pre-flight:
