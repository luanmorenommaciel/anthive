# p1 — `swarm scan`

> **Why this is step 1 of the CLI:** it's the simplest command to build, exercises the p0 contracts, and once it works you immediately see the value. Pointing `swarm scan` at ACT's `tasks/` should correctly list today's ready work.

---

## Goal

Implement the `swarm scan` subcommand: reads `tasks/**/*.md` in a repo, classifies each task by status + dependencies, emits a `ReadyList` JSON object.

Output formats:
- Default: styled Rich table (human-readable)
- `--json`: machine-readable ReadyList JSON to stdout
- `--plain`: ASCII table (CI/logging friendly, no ANSI)

---

## Success criteria

- [ ] `swarm scan` run from ACT repo root returns today's real state:
  - Ready: any task with `status: ready` and no unresolved `depends_on`
  - Blocked: any task with `status: blocked` or unresolved deps
  - In-progress: any task whose ID matches an active `logs/sessions/*.md`
  - Done: any task whose ID matches a merged PR in `logs/merge-queue.md`
- [ ] `swarm scan --json` output validates against `ReadyList` Pydantic model from p0
- [ ] `swarm scan --plain` renders without ANSI (for CI)
- [ ] Conflict detection: if two ready tasks touch the same file, flag under `conflicts`
- [ ] Handles ACT's current mix: per-phase files (`phase-*.md`), rollup (`backlog.md` with multiple `### B-*` blocks), nested (`agentswarm/*.md`)
- [ ] Fast: <500ms for ACT's ~14 task files
- [ ] `--watch` flag re-scans every 2s (live mode)
- [ ] Pytest suite: 10+ tests covering classification, dep resolution, conflicts

## Implementation sketch

```python
# agentswarm/scanner.py

from pathlib import Path
from .schemas import TaskFrontmatter, ReadyList, ReadyListEntry, BlockedEntry, parse_task_doc

def scan(repo_root: Path, tasks_dir: Path = Path("tasks")) -> ReadyList:
    """Scan tasks/**/*.md for task frontmatter, classify by status + dependencies."""
    tasks_path = repo_root / tasks_dir
    all_tasks: dict[str, TaskFrontmatter] = {}

    # 1. Discover — any .md file with a `id:` field in frontmatter is a task
    for md in tasks_path.rglob("*.md"):
        if md.name.startswith("_") or "archive" in md.parts:
            continue
        fm = parse_task_doc(md)
        if fm is None or not fm.id:
            continue
        all_tasks[fm.id] = fm

    # 2. Also scan backlog.md for B-* sub-tasks (today's pattern)
    backlog = tasks_path / "backlog.md"
    if backlog.exists():
        for sub in parse_backlog_blocks(backlog):
            all_tasks[sub.id] = sub

    # 3. Detect in-progress from logs/sessions/
    in_progress_ids = set()
    sessions_dir = repo_root / "logs" / "sessions"
    if sessions_dir.exists():
        for s in sessions_dir.glob("*.md"):
            if s.name == "_template.md":
                continue
            log = parse_session_log(s)
            if log.status not in ("MERGED",):
                in_progress_ids.add(log.task_id)

    # 4. Detect done from merge-queue.md
    done_ids = set()
    mq = repo_root / "logs" / "merge-queue.md"
    if mq.exists():
        for row in parse_merge_queue(mq):
            if row.merged:
                done_ids.add(row.task_id)

    # 5. Classify
    ready, blocked, in_progress, done = [], [], [], []
    for tid, t in all_tasks.items():
        if tid in done_ids:
            done.append({"id": tid, ...})
        elif tid in in_progress_ids:
            in_progress.append({"id": tid, ...})
        elif t.status == "blocked" or has_unresolved_deps(t, all_tasks, done_ids):
            blocked.append(BlockedEntry(id=tid, blocked_by=[...], reason="..."))
        elif t.status == "ready":
            ready.append(ReadyListEntry(id=tid, ...))

    # 6. Detect path conflicts among ready tasks
    conflicts = detect_conflicts(ready)

    return ReadyList(
        scanned_at=datetime.now(timezone.utc),
        repo_root=str(repo_root),
        tasks_dir=str(tasks_dir),
        ready=ready,
        blocked=blocked,
        in_progress=in_progress,
        done=done,
        conflicts=conflicts,
    )
```

### CLI (Typer)

```python
# agentswarm/cli.py

import typer
from rich.console import Console
from rich.table import Table

app = typer.Typer()
console = Console()

@app.command()
def scan(
    json_out: bool = typer.Option(False, "--json", help="JSON output"),
    plain: bool = typer.Option(False, "--plain", help="No ANSI/colors"),
    watch: bool = typer.Option(False, "--watch", help="Re-scan every 2s"),
):
    """Scan tasks/ and show what's ready to work on."""
    from .scanner import scan as do_scan
    result = do_scan(Path.cwd())

    if json_out:
        typer.echo(result.model_dump_json(indent=2))
        return

    # Rich table output
    table = Table(title=f"agentswarm scan — {len(result.ready)} ready / {len(result.blocked)} blocked")
    table.add_column("ID", style="cyan")
    table.add_column("Title")
    table.add_column("Status", justify="center")
    table.add_column("Effort", justify="center")
    table.add_column("Budget", justify="right")
    table.add_column("Agent")

    for r in result.ready:
        table.add_row(r.id, r.title, "[green]READY[/]", r.effort, f"${r.budget_usd}", r.agent)
    for b in result.blocked:
        table.add_row(b.id, "...", "[yellow]BLOCKED[/]", "-", "-", "-")
    # etc.

    console.print(table)

    if result.conflicts:
        console.print("\n[red]⚠ Path conflicts detected:[/]")
        for c in result.conflicts:
            console.print(f"  {' + '.join(c.task_ids)} → {c.paths[0]}")
```

## Backwards-compatibility note

ACT's current task files **don't yet have frontmatter** (they're human-authored markdown). `swarm scan` needs to handle both:

1. Files with proper p0 frontmatter → parsed strictly
2. Files without frontmatter → skipped gracefully (log warning, don't error)

This lets `agentswarm` ship against ACT's current docs without requiring a big migration. Users can add frontmatter incrementally.
