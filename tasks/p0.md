# p0 — Contracts (frontmatter schema + handoff formats)

> **Why first:** every other unit reads or writes one of these schemas. If the contracts are wrong, every downstream unit has to change. Define once, test hard, don't touch again.

---

## Goal

Freeze the four file-format contracts that the 8-unit architecture depends on:

1. **Task frontmatter schema** — the YAML block every task doc carries
2. **Ready-list JSON** — output of `swarm scan`
3. **Session log frontmatter** — the state-file format `swarm dispatch` writes
4. **Merge-queue row format** — what `swarm dispatch` appends and `swarm merge` consumes

All four must be: machine-parseable without an LLM, human-readable, git-diff-friendly, extensible (extra fields don't break readers).

---

## Success criteria

- [ ] `agentswarm/schemas.py` defines Pydantic models for all four contracts
- [ ] `tasks/agentswarm/contracts/task.example.md` shows a valid task doc

## The four contracts

### Contract 1 — Task frontmatter

Every `tasks/**/*.md` task doc carries a YAML frontmatter block. `swarm scan` parses these to build the ready list.

```yaml
---
# Required
id: T-20260424-card-violator-fix      # stable unique ID
title: "Fix card_validator alias bug"  # human-readable one-liner
status: ready                          # ready | blocked | in_progress | done
effort: S                              # XS | S | M | L | XL
budget_usd: 0                          # pre-flight cost estimate (0 = uses Max 20x)
agent: llm-evaluator-designer          # must exist at .claude/agents/**/<name>.md
depends_on: []                         # list of task IDs that must be done first
touches_paths:                          # paths this task will modify (conflict detection)
  - benchmark/card_validator.py
  - benchmark/tests/test_card_validator.py

# Optional
source: notes/2026-04-24-standup.md#L47  # provenance (meeting transcript line, etc.)
created: 2026-04-24T16:30:00-0300
prefer_model: sonnet                     # opus | sonnet | haiku | null
mode: local                              # local | cloud | null (uses config default)
max_turns: 50                            # cap on agent turns
tags: ["validator", "tech-debt"]
---
```

The body of the task doc (below the frontmatter) is the **specification** — what success looks like, tasks to complete, do-not-touch list. Same shape as today's `tasks/phase-*.md`.

### Contract 3 — Session log frontmatter

Same shape as today's `logs/sessions/_template.md`, promoted to a proper schema:

```yaml
---
session_id: s5-card-violator-fix
task_id: T-20260424-card-violator-fix
slug: card-violator-fix
branch: session/s5-card-violator-fix
worktree: worktrees/s5-card-violator-fix
container: swarm-s5                   # tmux pane name for local, or Managed Agents session ID for cloud
mode: local                           # local | cloud
forked_from_sha: abc123def456
created: 2026-04-24T17:45:00-0300
status: COOKING                       # INIT | COOKING | CHECKPOINT | READY-TO-MERGE | MERGED | BLOCKED
last_heartbeat: 2026-04-24T17:48:00-0300
last_note: "starting T4B.1.1 alias-bug fix (TDD)"
primary_agent: llm-evaluator-designer
model: sonnet-4.6
budget_usd: 0                         # cap (0 = no cap, uses subscription)
spent_usd: 0.08                        # live spend from Langfuse
tokens_in: 12430
tokens_out: 3400
touches_paths:
  - benchmark/card_validator.py
exit_check: "validator alias fix green (9/9 tests)"
pr_url: ""                            # filled when READY-TO-MERGE
langfuse_trace_url: "https://langfuse.example.com/trace/abc-123"
---

# Session s5-card-violator-fix

**Purpose:** (auto-filled by `swarm compose` from task doc)

## Timeline

- **2026-04-24T17:45:00-0300** · `INIT` · scaffolded by swarm dispatch
- **2026-04-24T17:46:00-0300** · `COOKING` · starting T4B.1.1 alias-bug fix (TDD)
```

### Contract 4 — Merge queue row

Appended by `swarm dispatch` when session hits `READY-TO-MERGE`. Consumed by `swarm merge`.

```markdown
- [ ] s5-card-violator-fix · PR #23 · touches: benchmark/card_validator.py, benchmark/tests/test_card_validator.py · depends-on: none · exit_check: 9/9 tests pass on main after merge · spent: $0.08 · langfuse: abc-123
```

Fields (separator: ` · `):
1. checkbox (ticked when merged)
2. session name
3. PR reference (if pushed)
4. `touches:` comma-separated paths
5. `depends-on:` comma-separated session names (or `none`)
6. `exit_check:` one-liner
7. `spent:` USD from Langfuse
8. `langfuse:` trace ID
