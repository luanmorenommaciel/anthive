# p7 — `swarm capture`

> **Why this is the grand-finale unit:** this closes the speech-to-commit loop. Krisp already captures the meeting and drops a transcript in `notes/`. Your `meeting-analyst` agent already exists in `.claude/agents/dev/`. p7 hooks them together so a new meeting automatically produces a curated task backlog.

---

## Goal

Implement `swarm capture`:

1. **Watch mode** (`swarm capture --watch`) — file-watcher on `notes/` directory; when a new transcript lands, automatically invoke the meeting-analyst agent and route its output to `tasks/pending/`
2. **One-shot mode** (`swarm capture <transcript.md>`) — process a specific file
3. **Review mode** (`swarm capture --review`) — interactive questionary UI to move `tasks/pending/*` into `tasks/` proper (with edits)

The full pipeline becomes:

```
Krisp meeting → notes/2026-04-25-standup.md
              → (watcher fires)
              → meeting-analyst agent invoked
              → tasks/pending/T-*.md written with p0 frontmatter contract
              → human review (optional, via `swarm capture --review`)
              → tasks/backlog.md rows appended (or left in pending/)
              → swarm scan picks them up
              → swarm dispatch runs them
```

---

## Success criteria

- [ ] File-watcher on `notes/` (via `watchdog`) triggers meeting-analyst on new `.md` files
- [ ] `swarm capture <file>` processes a single transcript end-to-end
- [ ] Output lands in `tasks/pending/T-<date>-<slug>.md` with valid p0 frontmatter
- [ ] Each pending task's `source:` frontmatter field points back to the transcript line that generated it (provenance)
- [ ] Over-generation protection: `tasks/pending/` is a staging area; nothing auto-flows into `tasks/backlog.md` without review
- [ ] `swarm capture --review` launches questionary: pick which pending tasks to promote, edit them before promoting
- [ ] Promoted tasks get moved from `tasks/pending/` to `tasks/backlog.md` (as a new `### B-*` block) or `tasks/<slug>.md` standalone files
- [ ] meeting-analyst agent is invoked via Claude Agent SDK `query(agents=...)` — stays in-process, no tmux
- [ ] Cost of capture is tracked in Langfuse (typically < $0.10 per transcript)
- [ ] Pytest: 6+ tests covering watcher, parse, review UI mocked

## Implementation sketch

```python
# agentswarm/capture.py

import asyncio
from pathlib import Path
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
from claude_agent_sdk import query, ClaudeAgentOptions, AgentDefinition
from .schemas import TaskFrontmatter
from .observability import session_span

PENDING_DIR = Path("tasks/pending")

async def process_transcript(transcript_path: Path, repo_root: Path) -> list[Path]:
    """Invoke meeting-analyst agent on a transcript; write output to tasks/pending/."""

    transcript_text = transcript_path.read_text()

    # Load the meeting-analyst agent definition
    agent_md = repo_root / ".claude/agents/dev/meeting-analyst.md"
    agent_def = parse_agent_md(agent_md)  # frontmatter + body

    prompt = f"""You are processing this meeting transcript:

---
PATH: {transcript_path.relative_to(repo_root)}
---

{transcript_text}

---

Your task: extract actionable backlog items from this transcript.

For each action item / decision / technical ask found, write ONE markdown file at
`tasks/pending/T-<YYYYMMDD>-<slug>.md` with this exact frontmatter:

---
id: T-<date>-<slug>
title: "<one-line title>"
status: ready
effort: XS | S | M | L | XL
budget_usd: <estimate, 0 if uses Max>
agent: <pick from .claude/agents/**>
depends_on: []
touches_paths: []
source: notes/<transcript>.md#L<line>
created: <iso timestamp>
---

Then the body with:
- **Why:** context from the meeting
- **Success criteria:** testable outcomes
- **Questions raised:** anything ambiguous

Write one file per item. Do NOT append to tasks/backlog.md — that's the user's
curation step. Only write to tasks/pending/.

Target 3-10 items per 30-min meeting. Quality over quantity. Skip chatter,
philosophical discussions, and anything already tracked."""

    pending_before = set(PENDING_DIR.glob("T-*.md")) if PENDING_DIR.exists() else set()

    with session_span(
        session_id=f"capture-{transcript_path.stem}",
        task_id="capture",
        agent="meeting-analyst",
        mode="capture",
    ):
        async for message in query(
            prompt=prompt,
            options=ClaudeAgentOptions(
                allowed_tools=["Read", "Write", "Grep", "Glob"],
                agents={
                    "meeting-analyst": AgentDefinition(
                        description=agent_def.description,
                        prompt=agent_def.body,
                        tools=["Read", "Write", "Grep", "Glob"],
                        model="sonnet",  # capture doesn't need Opus
                    ),
                },
            ),
        ):
            pass  # consume stream

    pending_after = set(PENDING_DIR.glob("T-*.md")) if PENDING_DIR.exists() else set()
    new_files = sorted(pending_after - pending_before)
    return new_files


class NotesHandler(FileSystemEventHandler):
    def __init__(self, repo_root: Path):
        self.repo_root = repo_root

    def on_created(self, event):
        if event.is_directory or not event.src_path.endswith(".md"):
            return
        asyncio.run(process_transcript(Path(event.src_path), self.repo_root))


def watch(repo_root: Path, notes_dir: Path = Path("notes")):
    """Start file-watcher on notes/ directory."""
    observer = Observer()
    handler = NotesHandler(repo_root)
    observer.schedule(handler, str(repo_root / notes_dir), recursive=False)
    observer.start()
    console.print(f"[green]Watching[/] {notes_dir}/ for new transcripts. Ctrl-C to stop.")
    try:
        while True:
            time.sleep(1)
    except KeyboardInterrupt:
        observer.stop()
    observer.join()


def interactive_review(repo_root: Path):
    """Launch questionary UI to review/promote pending tasks."""
    import questionary

    pending = sorted((repo_root / PENDING_DIR).glob("T-*.md"))
    if not pending:
        console.print("[dim]No pending tasks to review.[/]")
        return

    # Render each task as a choice
    choices = []
    for p in pending:
        fm = parse_task_doc(p)
        if not fm:
            continue
        label = f"{fm.id} — {fm.title}  [{fm.effort}]"
        choices.append(questionary.Choice(label, value=p))

    selected = questionary.checkbox(
        "Select tasks to promote from pending → backlog:",
        choices=choices,
    ).ask()

    for p in selected:
        fm = parse_task_doc(p)
        # Option 1: append to backlog.md as a ### B-* block
        # Option 2: move to tasks/<slug>.md as standalone
        action = questionary.select(
            f"Promote {fm.id} as:",
            choices=["standalone file (tasks/<slug>.md)", "backlog.md entry", "skip"],
        ).ask()
        if action.startswith("standalone"):
            dest = repo_root / "tasks" / f"{fm.id}.md"
            p.rename(dest)
            console.print(f"✓ [green]{dest}[/]")
        elif action.startswith("backlog"):
            append_to_backlog(repo_root / "tasks/backlog.md", p.read_text())
            p.unlink()
            console.print(f"✓ appended to backlog.md")
        # skip: leave in pending/
```

### CLI

```python
@app.command()
def capture(
    path: str = typer.Argument(None, help="Transcript file (one-shot)"),
    watch: bool = typer.Option(False, "--watch"),
    review: bool = typer.Option(False, "--review"),
):
    """Capture meeting transcripts into task backlog."""
    from .capture import process_transcript, watch as do_watch, interactive_review

    if review:
        interactive_review(Path.cwd())
        return

    if watch:
        do_watch(Path.cwd())
        return

    if path:
        new_files = asyncio.run(process_transcript(Path(path), Path.cwd()))
        console.print(f"[green]Captured {len(new_files)} pending task(s):[/]")
        for f in new_files:
            console.print(f"  • {f}")
        console.print(f"\n[dim]Run [cyan]swarm capture --review[/] to curate and promote.[/]")
        return

    typer.echo("Must specify: <path>, --watch, or --review", err=True)
    raise typer.Exit(2)
```

## Anti-patterns

- **Auto-promoting to backlog.md** — DON'T. Over-generation is the #1 risk. Human curates.
- **Parsing the transcript ourselves** — DON'T. That's meeting-analyst's job. `capture` is the orchestrator.
- **Requiring a Krisp integration** — DON'T. File-watcher on `notes/` is provider-agnostic. Works with any transcript source (Otter, Granola, hand-written).
- **Editing tasks in-place via the CLI** — DON'T. If user wants to edit, `$EDITOR` opens the file; capture just moves it.

---

## Do-not-touch list

- `tasks/backlog.md` — only appended via `append_to_backlog()`; never rewritten
- `tasks/*.md` existing files — never modified by capture
- `.claude/agents/dev/meeting-analyst.md` — read-only reference
- `notes/*.md` — read-only input
