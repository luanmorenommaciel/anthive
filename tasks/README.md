## The product in 3 lines

```
swarm scan         → what's ready to work on?
swarm dispatch     → spawn parallel sessions (local via Max 20x, or cloud via Managed Agents)
swarm watch        → live dashboard with cost/tokens/status, backed by Langfuse+OTEL
```

Plus `swarm merge` to reconcile PRs and `swarm capture` to hook meeting transcripts into the backlog.

## Decisions already locked (reference the research in PLAN.md)

- **Name:** `agentswarm`
- **Language:** Python
- **CLI stack:** Typer + Rich + questionary (Textual reserved for `watch` if it grows)
- **Observability:** Langfuse (self-hosted, MIT) + Claude Code's native OTEL exporter
- **Local auth:** OAuth via Claude Max 20x (free at the margin)
- **Cloud auth:** Console API key via Managed Agents ($0.08/session-hour + standard token cost)
- **Handoff format:** Markdown files with YAML frontmatter
- **Git model:** one worktree + one branch + one PR per session
- **Default mode:** local (don't burn API credits unnecessarily)

---

## What's *not* being built

- A GUI (TUI only via Textual for `watch`)
- A SaaS (self-hosted Langfuse + your own API key)
- Agent teams-style inter-session messaging (out of scope; would require rework)
- Its own LLM provider abstraction (Claude only at M1; OpenAI/etc. post-M4)
- A replacement for Claude Code (we wrap it, we don't replace it)

## Where today's ACT scripts fit

Today's `scripts/session_init.sh`, `session_heartbeat.sh`, `session_finish.sh` and `logs/` structure are the **reference implementation** for M1. Every bash script becomes a Python module; every log file stays in the same format (so existing ACT fleet data is immediately readable by `swarm`).

Files to port:

| Today | → | agentswarm M1 |
|---|---|---|
| `scripts/session_init.sh` | → | `agentswarm/scaffolder.py` (used by `swarm dispatch --local`) |
| `scripts/session_heartbeat.sh` | → | `agentswarm/state.py` (reads/writes log frontmatter) |
| `scripts/session_finish.sh` | → | `agentswarm/finisher.py` (used by `swarm merge`) |
| `logs/sessions/_template.md` | → | kept as-is (output format unchanged) |
| `logs/PROMPTS.md` | → | obsolete (auto-generated by `swarm compose`) |
| `logs/merge-queue.md` | → | kept as-is (same format, now auto-maintained) |

The port is effort-preserving: we don't throw out today's work, we make it reusable.
